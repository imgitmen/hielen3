    12 Maggio 2025

    • modifcata la vista v_feature_v2 aggiungengo flabel e ftype tra i campi ritornati

    • modificato in config/hielen.json la definizione delle chiavi di features_info_v2 ("feature","context","label") e di features_parameters_headers_v2 ("series","feature","param","ordinal","flabel","context")

    • modifcato modulo hielen3.series: Nel costruttore HSeries possibile passare all'argomento uuid un dict con queste chiavi

    ["series","feature","param","ordinal","flabel","context"]

    nessuna chiave obbligatoria

    dove: series l'uuid della serie se conosciuto ( si pu usare direttamente l'uuid al posto del dict ) feature l'uuid di una feature a cui la series fa riferimento (se conosciuto) param" il parametro della feature che individua la series ordinal l'ordinale associato alla feature che individua la series flabel il label del parametro associato alla feature a cui la series fa riferimanto context* un contesto appartenete alle famiglia di contesti a cui associata la feature di cui la series ne rappresenta un parametro

    es.: {"series":"26ba180b-2c6b-43df-874d-7b7e800a427b"} oppure solo "26ba180b-2c6b-43df-874d-7b7e800a427b" {"feature":"09334034-68a6-45da-91a8-8e86035111bb","ordinal":2} {"feature":"09334034-68a6-45da-91a8-8e86035111bb","param":"Y"} {"context":"67644aa916b77","flabel":"MG01","ordinal":2}

    tutti questi casi individuano univocamente una serie. Nel caso venga passato "flabel" opprtuno, ma non obbligatorio fornire anche il "context"

    questo dict viene passato direttamente al costruttore dell'oggetto python HSeries e si possono avere tre casi:

    1) la richiesta non ritorna alcun series uuid -> KeyError("Non trovato") 2) la richiesta ritorna un solo series uuid -> ok, la serie viene recuparata 3) la richiesta ritorna pi di un series uuid -> KeyError("Ambiguo")

    • modificato modulo hielen3.feature. Nella funzione retrive e retive_label inserito l'argomento context

    • modifcato modulo hielen3.ext.feature_instrument_importved.helpe: per permettere la configurazione dei parametri attravereso flabel e context

    • modificato modulo hielen3.contextmanager: moficate le funzioni family e ascendants in modo da ritornare errore nel caso in cui vengano fornito SOLO contesti non validi (non presenti nel db)

    • modificato hielen3.ext.feature_logger_loadsensing.helper per coprire il caso in cui il logger non invii la colonna della temperatura interna

    • modificato lo script di configurazione (non ancora nel git)

    Come funziona ColorMap:

    Incontrando il campo definito come "ColorMap" all'interno di una risposta di actionschemata, il front-end aprir l'interfaccia di scelta dei colori. Il risultato dell'interazione dovr essere sintetizzato in un json da fronire come parametro all'azione corrispondente. Il json dovr essere nella forma di un array di lunghezza variabile di COPPIE (valore,colore):

    [
    [
        valore,
        colore
    ],
    [
        valore,
        colore,
    ]
    ...
]

    dove:

    valore un numero di qualsiasi genere con punto decimale

    colore una stringa di colore hex

    esempio:

    [
    [
        -0.25,
        '#FFFF00'
    ],
    [
        -0.125,
        '#00FF00'
    ],
    [
        0,
        '#0000FF'
    ],
    [
        0.5,
        '#FF00FF'
    ],
    [
        1,
        '#AEBD00'
    ],
                                            ]

    Un array del genere verrebbe inteso in questo modo:

    [-inf,-0.25) :

    [-0.25,-0.125) : -->

    [-0.125,0.0) : -->

    [0.0,0.5) : -->

    [0.5,1.0) : -->

    [1.0,inf) :

    L'informazione sulla colormap pu essere associata ad ongi singola serire dati e pu essere fornita in front-end su richiesta: allo scopo L'API features stata modificata in modo da fornire la colormap insieme con i parametri delle features specificando l'opzione info=parameters,...

    esempio (vedi chiamata):

    {
    ...,
    "parameters": [
        {
            "series": "2b6c9efb4e83f64e757ce862f8b6c2e0",

                [
                    -0.25,
                    "#FFFF00",
                    0
                ],

                    -0.125,

                    10

                    20

    ...

    dove ogni cmap un array ordinato di triplette utile alla creazione della relativa colorbar. Ogni tipletta fornisce questa infromazione:

    cmap[x][0]: etichetta del valore, possibilmente da piazzare sulla colorbar alla giusta posizione.
